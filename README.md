# r1Keyfolio

Пакетный сканер приватных ключей и адресов по нескольким блокчейнам.

Вставляешь список `hex` / `WIF` / адресов — приложение локально в браузере
выводит адреса и проверяет балансы в публичных сетях. Приватные ключи никуда
не отправляются: вся деривация идёт на клиенте, наружу уходят только адреса.

## Возможности

- Ввод построчно или загрузка `.txt` пачками
- Поддержка входа: приватный ключ `hex` (64), `WIF`, готовый адрес
- Режимы **NORMAL**, **BRAIN** и **BOTH** для обычной, brainwallet или
  одновременной обработки каждой строки
- Деривация четырёх BTC-адресов: P2PKH compressed/uncompressed,
  P2SH-P2WPKH (script) и native SegWit (bc1)
- Деривация адресов из ключа: LTC, DOGE, DASH, ZEC,
  ETH/BSC/Polygon/Arbitrum/Optimism/Base/Avalanche, TRON
- Проверка баланса, кол-ва транзакций и признака «живой» адрес
- Фильтры (funded / alive / empty / err), поиск, экспорт CSV
- Несколько бесплатных API-провайдеров на сеть с автоматическим фолбэком
- Пер-хостовый троттлинг, экспоненциальный бэкофф и учёт `Retry-After`

## Провайдеры и лимиты

Чтобы не упираться в лимиты публичных API, для каждой сети запросы идут по
цепочке провайдеров — если первый вернул лимит или ошибку, автоматически
пробуется следующий:

| Сеть | Провайдеры (в порядке приоритета) |
|------|-----------------------------------|
| BTC  | mempool.space → blockstream.info → BlockCypher |
| LTC  | litecoinspace.org → BlockCypher |
| DOGE | Blockchair → BlockCypher |
| DASH | Blockchair → Dash Insight |

Дополнительно:

- **Пер-хостовый троттлинг** — запросы к одному домену разнесены по времени,
  чтобы не ловить `429`.
- **Бэкофф с `Retry-After`** — при `402/429/503` ожидание берётся из заголовка
  ответа, иначе экспоненциально растёт.
- **Статус `⏳` (skipped)** — если все провайдеры сети временно упёрлись в лимит,
  ячейка помечается как пропущенная (не ошибка); можно перезапустить скан позже.

## Brainwallet

Классическая схема: строка ввода используется как UTF-8 текст, из неё локально
получается ключ `SHA256(input)`, дальше обычная деривация адресов по выбранным
сетям. Поэтому в этом режиме можно обрабатывать и passphrase, и строки с
hex-хэшами.

1. Выбери режим в панели ввода:
   - **NORMAL** — интерпретировать строки как `hex`, `WIF` или адреса
   - **BRAIN** — пропустить каждую строку через SHA-256
   - **BOTH** — создать для каждой строки обе версии и проверить обе
2. Вставь фразы построчно или загрузи `.txt` через **TXT**
3. Нажми **SCAN**

Пример содержимого файла:

```
correct horse battery staple
my secret phrase
password
```

Строки, начинающиеся с `#` или `//`, и пустые строки пропускаются.

> Brainwallet слабее обычного случайного ключа: простые фразы легко
> перебираются. Используй только для проверки своих старых passphrase.

## Стек

React + TypeScript + Vite, `@noble/*` и `@scure/*` для крипто, `ethers` для EVM.

## Быстрый старт (dev)

```bash
npm install
npm run dev
```

Откроется на `http://localhost:5173`. В dev-режиме запросы к API идут через
встроенный прокси Vite (без CORS).

## Docker (production)

```bash
docker compose up --build -d
```

Сайт: `http://localhost:8085`. Внутри — nginx, который раздаёт статику и
проксирует API сетей (`/x/...`), чтобы обойти CORS.

## Сборка

```bash
npm run build
```

## Ответственное использование

Инструмент предназначен для проверки **своих** ключей и адресов. Не используй
его для доступа к чужим средствам.
