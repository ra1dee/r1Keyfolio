# r1Keyfolio

Пакетный сканер приватных ключей и адресов по нескольким блокчейнам.

Вставляешь список `hex` / `WIF` / адресов — приложение локально в браузере
выводит адреса и проверяет балансы в публичных сетях. Приватные ключи никуда
не отправляются: вся деривация идёт на клиенте, наружу уходят только адреса.

## Возможности

- Ввод построчно или загрузка `.txt` пачками
- Поддержка входа: приватный ключ `hex` (64), `WIF`, готовый адрес
- Режим **BRAIN** (brainwallet): `SHA256(UTF-8 passphrase)` → private key
- Деривация адресов из ключа: BTC (legacy + SegWit), LTC, DOGE, DASH, ZEC,
  ETH/BSC/Polygon/Arbitrum/Optimism/Base/Avalanche, TRON
- Проверка баланса, кол-ва транзакций и признака «живой» адрес
- Фильтры (funded / alive / empty / err), поиск, экспорт CSV
- Retry и параллельные запросы

## Brainwallet

Классическая схема: каждая строка ввода — passphrase, из неё локально
получается ключ `SHA256(passphrase)`, дальше обычная деривация адресов по
выбранным сетям.

1. Включи кнопку **BRAIN** в панели ввода
2. Вставь фразы построчно или загрузи `.txt` через **TXT**
3. Нажми **SCAN**

Пример содержимого файла:

```
correct horse battery staple
my secret phrase
password
```

Строки, начинающиеся с `#` или `//`, и пустые строки пропускаются.

> Brainwallet слабее обычного случайного ключа: простые фразы легко
> перебираются. Используй только для проверки своих старых passphrase.

## Стек

React + TypeScript + Vite, `@noble/*` и `@scure/*` для крипто, `ethers` для EVM.

## Быстрый старт (dev)

```bash
npm install
npm run dev
```

Откроется на `http://localhost:5173`. В dev-режиме запросы к API идут через
встроенный прокси Vite (без CORS).

## Docker (production)

```bash
docker compose up --build -d
```

Сайт: `http://localhost:8085`. Внутри — nginx, который раздаёт статику и
проксирует API сетей (`/x/...`), чтобы обойти CORS.

## Сборка

```bash
npm run build
```

## Ответственное использование

Инструмент предназначен для проверки **своих** ключей и адресов. Не используй
его для доступа к чужим средствам.
